3 - Criando o Product Backlog

Objetivo

Aprender a criar e organizar o Product Backlog, entendendo como ele orienta o trabalho do time e define o que deve ser desenvolvido.

O que é o Product Backlog

O Product Backlog é uma lista priorizada de tudo o que será desenvolvido no produto. Ele contém as necessidades do cliente e as funcionalidades desejadas, descritas de forma simples e compreensível.

Cada item do backlog é chamado de User Story (história de usuário), que descreve uma funcionalidade do ponto de vista de quem vai usá-la.

Formato de uma User Story

Uma história de usuário geralmente segue o formato:

“Como [tipo de usuário], quero [função] para [benefício].”

Exemplos:

  • “Como cliente, quero criar uma conta para acessar meu perfil.”
  • “Como usuário, quero receber notificações para acompanhar meus pedidos.”
  • “Como administrador, quero visualizar relatórios de vendas para planejar ações.”

Priorização do Backlog

Nem todas as histórias têm a mesma importância. O Product Owner deve priorizar o backlog, colocando no topo os itens que entregam mais valor ao cliente.

O backlog é dinâmico: ele muda e evolui conforme o projeto avança e novas ideias surgem.

Boas práticas

  • Escreva histórias curtas e claras.
  • Evite termos técnicos demais.
  • Divida histórias grandes em partes menores.
  • Revise o backlog com frequência junto ao time.

Atividade

Atividade 3 – Criando seu Product Backlog:

1. Retome o projeto escolhido na primeira atividade.
2. Crie de 5 a 8 histórias de usuário que representem as principais funcionalidades.
3. Organize as histórias em ordem de prioridade (da mais importante à menos urgente).
4. Verifique se cada história segue o formato:
“Como [usuário], quero [função] para [benefício].”

Resumo

  • O Product Backlog lista tudo que o produto precisa ter.
  • Cada item é descrito como uma User Story.
  • O Product Owner é responsável por manter o backlog priorizado e atualizado.
  • O backlog é um documento vivo, que evolui junto com o produto.
